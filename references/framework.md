# 家庭财务管理框架

## 1. 定位

这套框架不是炒股助手，而是家庭财务决策系统。它把家庭目标、现金流、安全垫、保险、房产、负债、资产配置、风险桶、标的研究、交易纪律和复盘记忆放进同一套流程。

核心问题不是"这个标的好不好"，而是：

1. 家庭资产负债表是否安全。
2. 未来 6-12 个月现金流是否安全。
3. 保险、房产、负债和专项资金是否构成硬约束。
4. 金融资产配置是否服务家庭目标。
5. 单次交易是否破坏风险桶、现金安全垫和行为纪律。
6. 每次建议是否有证据、边界、失效条件和复盘机制。

## 2. 最高原则

家庭安全 > 现金流 > 风险保障 > 资产配置 > 标的机会 > 执行价格。

这条原则的含义是：一个看起来很好的投资机会，如果会牺牲家庭安全垫、保险保障、专项资金、房贷压力或既定风险预算，就不是好机会。

## 4. 三档执行流程

| 档位 | 触发 | 输出 |
| --- | --- | --- |
| 轻流程 | 记录、已批准再平衡、小额且不改变风险桶的动作 | 操作记录 + 必要的资产表更新 |
| 标准流程 | 已研究过的标的、风险桶内替换、中等金额动作 | 决策 memo + 操作记录 + 复盘日期 |
| 重流程 | 新资产类别、新公司/基金、单笔较大、风险桶接近上限、用户情绪强 | 正式 memo / 专项报告 + 反方审查 + 压力测试 + 记忆写入 |

如果不确定档位，升一档处理。档位决定流程强度，不决定结论松紧。

## 5. 决策闸门

每次投资或配置建议按下面顺序走：

1. 数据日期：最新资产、负债、现金流、保险、持仓和市场价格是什么时候的。
2. 家庭安全：现金/类现金、安全垫、未来大额支出、收入稳定性、保险和负债。
3. 资金属性：这笔钱是短期资金、专项资金，还是长期投资资金。
4. 资产配置：当前大类资产是否偏离目标，新增动作是否帮助再平衡。
5. 风险桶：标的属于哪个风险桶，该桶是否低配、标配、超配。
6. 标的质量：公司、基金、债券、黄金、房产或其他资产的底层逻辑。
7. 估值/成本：当前价格隐含预期、费用率、替代方案和安全边际。
8. 行为风险：追涨、补仓冲动、亏损厌恶、熟悉度偏误、沉没成本。
9. 执行计划：金额、顺序、分批、触发条件、禁止动作。
10. 复盘机制：日期、指标、失效条件、记忆写入位置。

只要关键闸门未通过，输出"禁止 / 等待 / 需要补信息"，不要给强执行建议。

## 6. 资产和风险桶

推荐先按资产类别记录，再按风险桶管理。

常见资产类别：

- 现金和类现金
- 货币基金 / 短债 / 纯债
- 宽基权益
- 行业或主题权益
- 单一公司股票
- 海外资产
- 黄金或商品
- 房产
- 保险现金价值或保障计划
- 负债

常见风险桶：

- 家庭安全桶：应急金、未来 6-12 个月现金流、专项资金。
- 稳定收益桶：低波动债券、货币、现金管理。
- 核心增长桶：宽基权益、全球分散权益。
- 卫星风险桶：行业、主题、单票、集中资产。
- 防御对冲桶：黄金、外币资产、低相关资产。
- 非投资桶：自住房、刚性保险、不能动的钱。

风险桶的作用是防止同类风险在不同名称下重复加仓。

## 7. 记忆写入规则

不要把所有报告全文写进记忆。记忆只沉淀可复用结论。

| 写入层 | 写什么 | 不写什么 |
| --- | --- | --- |
| L1 | 家庭阶段、目标、稳定约束 | 每笔流水 |
| L2 | IPS、规则、阈值、工作流 | 临时市场观点 |
| L3 | 进行中任务、等待确认、阻塞 | 已完成旧任务细节 |
| L4 | 资产配置、公司、基金、行业的稳定认知 | 未验证传闻 |
| L5 | 复盘教训、行为偏差、长期反模式 | 单次价格波动 |

每条记忆建议格式：

```markdown
## YYYY-MM-DD - [标题]
- 来源：
- 事实/结论：
- 影响：
- 置信度：高 / 中 / 低
- 下次复核：
```

## 8. 信息不足时的输出

不要追问十几个问题。先列最小补齐清单：

```markdown
需要补信息：
1. 最新家庭资产负债表日期。
2. 本次资金来源和是否属于专项资金。
3. 当前现金安全垫目标与实际金额。
4. 标的代码、市场、已有持仓和本次拟操作金额。

在补齐前，只能输出观察建议，不能输出买入/卖出强建议。
```

## 9. 脱敏规则

制作模板、演示或可分享文件时：

- 使用 `{{HOUSEHOLD_NAME}}`、`{{BASE_CURRENCY}}`、`{{DATE}}` 等占位符；
- 不保留真实姓名、家庭成员细节、账户名、券商、银行卡、截图来源、金额、持仓数量、成本价、交易日期组合；
- 示例金额使用明显假数，如 `100,000`，并标注"示例"；
- 不复制 `.env`、`config.yaml`、cookie、token、OpenAPI 配置；
- 不把真实失败经历写成可识别故事，除非用户明确允许。
